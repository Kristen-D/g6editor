更新 - 2018年4月


初始访问策略添加

初始访问被添加到ATT和CK，并且一些技术被添加到执行中以涵盖PRE-ATT和CK内的启动和妥协技术。这些技术经过重构，以适应企业级的细节。

初始访问下添加了以下技术：

  • 驾车妥协
  • 利用面向公众的应用程序
  • 硬件增加
  • Spearphishing附件
  • 鱼叉式钓鱼链接
  • 通过服务进行鱼叉式网络钓鱼
  • 供应链妥协
  • 值得信赖的关系

以下现有技术交叉引用到初始访问：

  • 通过可移动媒体进行复制
  • 有效帐户

执行中添加了以下技术：

  • 利用客户端执行
  • 用户执行

其他变化

利用漏洞突破 - 通过添加初始访问，需要更清晰地定义软件开发行为。最初的Vulnerability漏洞技术被分为六种变体，专门用于个人策略。

  • 初始访问 - 利用面向公众的应用程序
  • 执行 - 客户执行的利用
  • 权限提升 - 权限提升的利用：维护最初的漏洞利用技术ID T1068
  • 横向运动 - 远程服务的开发
  • 防御逃避 - 防御逃避的剥削
  • 凭证访问 - 凭证访问的利用

软件平台 - 为软件对象添加了Windows，Linux和macOS标记。

什么是PRE-ATT和CK的新功能？

此版本弃用了Launch和Compromise策略以及属于它们的大多数技术。将来，ATT和CK 的初始访问和执行策略中的技术将涵盖这些TTP 。

  • 2个战术已被弃用：发布和妥协
  • 传播可移动媒体已从启动阶段转移到阶段功能
  • 技术信息收集中增加了一项新技术，即信息的Spearphishing
  • 此版本中已弃用了23种技术
  • PRE-ATT＆CK现在包括15种战术和151种技术
