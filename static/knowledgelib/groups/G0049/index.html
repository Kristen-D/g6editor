石油钻机

OilRig是一个伊朗起源的威胁组织，自2014年以来一直瞄准中东和国际受害者。该组织的目标是各种行业，包括金融，政府，能源，化工和电信，并且主要集中在中东地区。该组织似乎进行供应链攻击，利用组织之间的信任关系来攻击其主要目标。FireEye根据基础设施细节评估该组织代表伊朗政府工作，这些细节包含对伊朗的引用，伊朗基础设施的使用以及与民族国家利益一致的目标。[1] [2] [3] [4] [5] [6] 此组先前在两个不同的组APT34和OilRig下进行了跟踪，但由于额外报告的结合，对活动的重叠提供了更高的信心。

ID：G0049
别名：OilRig，Helix Kitten，APT34
贡献者：Robert Falcone; 布莱恩李

使用的技术

域ID名称使用
企业T1087帐户发现石油挖掘钻塔已经用完net user，net user /domain，net group “domain admins” /domain，并net group “Exchange Trusted Subsystem” /domain获得对受害者的帐户列表。[3]
企业T1119自动收集OilRig使用自动收集。[5]
企业T1110蛮力OilRig使用蛮力技术获取凭证。[7]
企业T1059命令行界面OilRig使用命令行界面执行。[6] [8] [5] [7]
企业T1223编译的HTML文件OilRig使用CHM有效负载加载并执行另一个恶意文件，一旦传递给受害者。[3]
企业T1003凭证倾销OilRig使用凭证转储工具（如Mimikatz和Lazagne）窃取登录到受感染系统和Outlook Web Access的帐户的凭据。[5] [7]
企业T1094自定义命令和控制协议OilRig已经为C2使用了自定义DNS隧道协议。[5]
企业T1140反混淆/解码文件或信息一个石油挖掘钻塔宏已运行PowerShell命令解码文件内容。OilRig还使用certutil解码受害者的base64编码文件。[6] [9] [8]
企业T1048对替代议定书的渗透OilRig通过FTP将数据与其主要的C2通道分开。[4]
企业T1133外部远程服务OilRig使用VPN，Citrix或OWA等远程服务来保持环境。[7]
企业T1008后备频道OilRig恶意软件ISMAgent如果无法通过HTTP访问C2服务器，则会回退到其DNS隧道机制。[10]
企业T1107文件删除OilRig在执行后删除了与其有效负载相关的文件。[6] [8]
企业T1066从工具中删除指标OilRig已测试恶意软件样本以确定AV检测，随后修改样本以确保AV逃避。[1]
企业T1056输入捕获OilRig使用了一种名为KEYPUNCH的键盘记录工具。[7]
企业T1046网络服务扫描OilRig使用公开的工具SoftPerfect Network Scanner以及一个名为GOLDIRONY的自定义工具来进行网络扫描。[7]
企业T1027混淆的文件或信息OilRig在其恶意软件中加密和编码数据，包括使用base64。[6] [11] [5]
企业T1201密码策略发现OilRig在脚本中使用net.exe net accounts /domain来查找域的密码策略。[12]
企业T1069权限组发现石油挖掘钻塔已经使用net group /domain，net localgroup administrators，net group “domain admins” /domain，并net group “Exchange Trusted Subsystem” /domain找到一个受害群体的权限设置。[3]
企业T1086电源外壳OilRig使用PowerShell脚本执行，包括使用宏来运行PowerShell命令来解码文件内容。[6] [9]
企业T1057流程发现OilRigtasklist在受害者的机器上运行。[3]
企业T1012查询注册表OilRig已经使用reg query “HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default”受害者来查询注册表。[3]
企业T1108冗余访问OilRig通过Web shell 使用RGDoor建立冗余访问。该组还使用收集的凭据来访问可访问Internet的Web资源，例如Outlook Web Access，这些资源可用于冗余访问。[5]
企业T1076远程桌面协议OilRig使用远程桌面协议进行横向移动。该小组还使用隧道工具将RDP隧道传输到环境中。[5] [7]
企业T1105远程文件复制OilRig可以将远程文件下载到受害者身上。[6]
企业T1021远程服务OilRig使用Putty访问受损系统。[5]
企业T1053预定任务OilRig创建了运行VBScript以在受害计算机上执行有效负载的计划任务。[8] [11]
企业T1113屏幕截图OilRig有一个名为CANDYKING的工具来捕获用户桌面的截图。[7]
企业T1064脚本OilRig has used various types of scripting for execution, including .bat and .vbs scripts. The group has also used macros to deliver malware such as QUADAGENT and OopsIE.[6][10][8][11]
EnterpriseT1193Spearphishing AttachmentOilRig has sent spearphising emails with malicious attachments to potential victims using compromised and/or spoofed email accounts.[8][11]
EnterpriseT1192Spearphishing LinkOilRig has sent spearphising emails with malicious links to potential victims.[8]
EnterpriseT1071Standard Application Layer ProtocolOilRig has used HTTP and DNS for C2. The group has also used the Plink utility and other tools to create tunnels to C2 servers.[5][7]
EnterpriseT1032Standard Cryptographic ProtocolOilRig used the Plink utility and other tools to create tunnels to C2 servers.[7]
EnterpriseT1082System Information DiscoveryOilRig has run hostname and systeminfo on a victim.[3][4]
EnterpriseT1016System Network Configuration DiscoveryOilRig has run ipconfig /all on a victim.[3][4]
EnterpriseT1049System Network Connections DiscoveryOilRig has used netstat -an on a victim to get a listing of network connections.[3]
EnterpriseT1033System Owner/User DiscoveryOilRig has run whoami on a victim.[3][4]
EnterpriseT1007System Service DiscoveryOilRig利用sc query受害者收集有关服务的信息。[3]
企业T1204用户执行OilRig提供了恶意链接和启用宏的文档，这些文档要求目标单击“启用内容”按钮以执行系统上的有效负载。[8] [11]
企业T1078有效帐户OilRig使用受损凭证访问受害网络上的其他系统。[5] [7]
企业T1100Web ShellOilRig经常使用Web shell来维护对受害者网络的访问。[5] [7]
企业T1047Windows Management InstrumentationOilRig使用WMI执行。[7]
